**프로젝트 실행 방법**
1. 프로젝트 클론
2. 라이브러리 설치 확인: 프로젝트 루트 디렉토리에서 npm install 명령어를 실행하여 모든 라이브러리가 제대로 설치되었는지 확인합니다.
- `npm install papaparse xlsx chart.js react-chartjs-2 chartjs-adapter-date-fns date-fns simple-statistics lodash chartjs-plugin-datalabels`
3. 개발 서버 실행: npm run dev 명령어로 Next.js 개발 서버를 실행합니다.
4. 접속 및 테스트: http://localhost:3000에 접속하여 다음을 테스트합니다.

**시각화 도구 테스트 방법**
1. sales_data.csv 또는 employee_performance.csv 파일을 업로드합니다.
2. 업로드 후 '저장된 데이터셋' 목록에 파일이 추가되고 자동으로 로드되는지 확인합니다.
3. X축, Y축, 차트 종류를 바꿔보며 시각화가 잘 되는지 확인합니다.
4. 'Date' 컬럼을 X축으로, 'SalesAmount'를 Y축으로 선택하고 '선 차트'로 변경하여 날짜별 추이가 잘 나오는지 확인합니다.
5. '데이터 필터링' 섹션에서 'Product' 컬럼을 선택하고 'Laptop'을 입력하여 필터링이 되는지 확인합니다.
6. '데이터 그룹화' 섹션에서 체크박스를 활성화하고 'Product' (그룹 기준), 'SalesAmount' (Y축 값), '합계' (집계 유형)를 선택하여 제품별 총 매출이 막대 차트로 표시되는지 확인합니다.
7. '차트 커스터마이징' 섹션에서 차트 제목, 축 제목, 색상, 데이터 라벨 표시 등을 바꿔보며 적용되는지 확인합니다.
8. 두 개의 숫자 컬럼 (예: employee_performance.csv에서 'PerformanceScore'와 'Salary(USD)')을 X, Y축으로 선택하고 차트 종류를 '산점도'로 변경하여 산점도가 그려지고 상관계수가 표시되는지 확인합니다.
9. 페이지를 새로고침하거나 브라우저를 닫았다가 다시 열었을 때, 로컬 스토리지에 저장된 데이터셋 목록과 마지막으로 선택했던 차트 설정이 유지되는지 확인합니다.
10. 데이터셋 삭제 버튼을 눌러 목록에서 제거되는지 확인합니다.

- 보안 관련해서는 현재 구조에서 클라이언트 측 데이터 처리가 주를 이루므로 직접적인 스크립트 삽입 위험은 낮지만, 항상 사용자 입력에 대한 검증과 출력 시 이스케이프 처리는 기본적인 원칙으로 염두에 두어야 합니다.

**인터넷으로 접속하지만 모든 데이터 처리와 저장을 로컬 스토리지에서만 활용하는 방식의 장점:**

- 강력한 데이터 프라이버시 및 보안: 사용자의 데이터가 당신의 서버나 어떤 외부 서버로도 전송되지 않습니다. 모든 작업이 사용자 브라우저 내에서 이루어지기 때문에, 민감한 개인 정보나 기업 비밀 데이터라도 유출될 위험이 현저히 낮습니다. 이는 특히 데이터 보안이 최우선인 시각화 도구에서 매우 강력한 강점입니다.
- 낮은 운영 비용: 서버, 데이터베이스, 백엔드 로직 등이 전혀 필요 없으므로, 사실상 호스팅 비용이 거의 들지 않습니다. 정적인 파일만 배포하면 되기 때문이죠.
- 빠른 응답 속도: 네트워크 통신이 없어 데이터 로드 및 처리 속도가 매우 빠릅니다.

**인터넷으로 접속하지만 모든 데이터 처리와 저장을 로컬 스토리지에서만 활용하는 방식의 잠재적 한계:**

- 로컬 스토리지 용량 제한: 브라우저마다 다르지만, 일반적으로 로컬 스토리지는 5MB에서 10MB 정도의 용량 제한이 있습니다. 이 용량을 초과하는 대용량 파일을 업로드하거나, 많은 파일을 로컬에 저장하려고 하면 오류가 발생할 수 있습니다.
- 데이터 지속성 (Persistence)의 한계:
사용자가 캐시를 지우거나, 다른 브라우저/다른 기기에서 접속하면 이전에 저장했던 데이터는 사라집니다.
같은 기기라도 시크릿 모드/게스트 모드에서는 데이터가 영구적으로 저장되지 않습니다.
- 복잡한 분석 및 대규모 데이터 처리의 제약:
브라우저의 자원(메모리, CPU)만으로 처리해야 하므로, 수십만 줄 이상의 매우 큰 데이터셋에 대한 복잡한 통계 분석이나 고급 머신러닝 기능 등을 구현하는 데는 한계가 있습니다.
simple-statistics, lodash 같은 라이브러리가 유용하지만, 서버 측 연산만큼 강력하지는 않습니다.
- 협업 기능 부재: 여러 사용자가 데이터를 공유하고 함께 작업하는 기능은 제공할 수 없습니다. 데이터는 각 사용자의 브라우저에만 존재합니다.
보안 (XSS): 앞서 언급했듯이, 사용자 입력값을 HTML로 직접 렌더링하는 기능을 추가한다면 XSS 취약점이 생길 수 있습니다. 현재는 Chart.js가 안전하게 처리하지만, 향후 기능 확장 시 항상 주의해야 합니다.
